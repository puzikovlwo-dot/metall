# Аксвил — Дашборд продаж (MVP)

Интерактивный дашборд отдела продаж компании «Аксвил» (металлопрокат).  
Стек: Python (pandas, numpy) + Dash + Plotly + Bootstrap.

---

## Функциональность

- **Ролевая модель:** Директор / Коммерческий директор / Менеджер (изоляция данных)
- **Фильтры:** период, тип клиента, категория товара, статус сделки
- **6 KPI-карточек:** выручка, выполнение плана, количество продаж, средний чек, динамика, клиенты (новые/старые)
- **Графики:** линейный (выручка + план), столбчатый (по менеджерам), круговая диаграмма (по категориям)
- **Таблица детализации** с сортировкой и пагинацией
- **База клиентов** (модальное окно с фильтрацией по роли и типу клиента)
- **Автоматические аналитические выводы**
- **Экспорт в CSV** с учётом текущих фильтров
- Адаптивный дизайн, корпоративный B2B-стиль

---

## Установка и запуск

### 1. Требования

- Python 3.10+
- Установленные зависимости:

```bash
pip install dash dash-bootstrap-components plotly pandas numpy
```

### 2. Генерация данных

```bash
python data_prep.py
```

Создаёт три CSV-файла:
- `sales.csv` — 2000 сделок (5 менеджеров, 4 категории, 4 статуса)
- `plans.csv` — месячные планы по менеджерам
- `clients.csv` — справочник клиентов (новый/старый, менеджер первой сделки)

### 3. Запуск дашборда

```bash
python app.py
```

Открой браузер: **http://127.0.0.1:8050**

## Данные

**sales** — 2000 строк, поля:
- Дата, Номер договора, ID клиента, Название клиента, Тип клиента (Опт/Розница)
- ID менеджера, ФИО менеджера, Категория (Трубы/Лист/Арматура/Метизы)
- Количество, Цена за ед., Сумма, Статус (Оплачено/В работе/Отменено/Отозвано)

**plans** — 30 строк (5 менеджеров × 6 месяцев), поле: Плановая сумма

**clients** — справочник клиентов со статусом (Новый/Старый) и статистикой сделок

## Особенности

- Статусы сделок зависят от комбинации (тип клиента × категория) — матрица вероятностей
- Сезонность продаж (повышение в марте и июне, спад в январе и мае)
- Специализация менеджеров по типам клиентов и категориям
- Pareto-распределение активности клиентов (20% клиентов = 80% продаж)
- План рассчитывается пропорционально дням в выбранном периоде
- При отсутствии данных KPI и графики показывают «—» / «Нет данных»
